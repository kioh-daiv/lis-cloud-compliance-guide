# 臨床検査部門システム（LIS）クラウド化 — 法規制対応と技術要件ガイド

検査・細菌・病理の臨床検査部門システム（LIS）をクラウドで実現するために満たすべき法規制を、**根拠条項・要件・技術スタック・残リスク**まで分解して整理したシングルページ HTML ガイド。あわせて「これで網羅できているか」を導出フレームワークで示し、第三者からの追加質問に答えられる構造にしている。

## 設計思想

法規制の個別列挙では網羅は証明できない（不存在の証明はできない）。本ガイドは 2 種類の証明を組み合わせる。

- **横の証明（網羅性）**：① データライフサイクル全段階 × ② 3省2ガイドラインを傘とする法体系階層、の 2 軸で機械的に洗い出す。対象/対象外を明示宣言し「漏れ」と「意図的除外」を分離。
- **縦の証明（深さ）**：各規制を 5 段で分解 ——**該当条項 → 満たすべき要件 → 技術スタック（原則＋AWS 実現例）→ なぜ担保できるか → 残リスク**。残リスクは「追加技術／運用／契約・SLA／受容」の 4 分類で引き取り先を明示。

## 構成（4 タブ）

- **必須の法規制（10）**：個人情報保護法、クラウド例外（第三者提供/委託の該当性）、医療法施行規則のサイバーセキュリティ確保義務＋3省2GL、インシデント報告ルート、検体検査の精度の確保、衛生検査所登録、感染症法、電子保存の三原則、記録の法定保存義務、薬機法（SaMD・病理診断の医行為性）
- **推奨する対応（7）**：ISMS（JIS Q 27001:2023 / ISO 27017・27018）、プライバシーマーク、ISMAP/ISMAP-LIU、次世代医療基盤法、標準規格（FHIR/JLAC11）、サイバーセキュリティ経営ガイドライン、JAHIS MDS/SDS
- **要件 × 技術スタック対応表**：21 の要件カテゴリ × 原則 × AWS 実現例 × 残リスク
- **トレーサビリティ表**：法規制 × 条項 × 要件 × 技術 × 残リスクの索引

## 検証プロセス

4 つの専門ドメイン（医療情報セキュリティ／個人情報・データ利活用／検査業務固有／クラウド認証・技術妥当性）でレビューし、条項番号・施行日・版番を一次資料で再照合した。主な確定修正：事業者向け GL の第2.0版（令和7年3月）への更新、次世代医療基盤法の題名改正、漏えい確報の60日枝、AWS HealthLake の国内非提供（自前 FHIR への差し替え）、S3 Object Lock のコンプライアンスモード明記、Macie の日本語 PII 制約。

## 注意

2026 年 5 月時点で公開されている公的情報に基づく一般整理。法規制・ガイドラインは改定され、条項番号も改正で変動しうる。一部の条項各号の細目は公的解説に基づく確認であり、実際の適用・顧客提示の前に原典・所管官庁・法務/薬事/セキュリティ専門家の確認を前提とする。技術スタックは一例で、AWS 以外のクラウドでも同等の原則を満たせばよい。
